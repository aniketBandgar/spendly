⏺ This is a student learning project — a Flask-based expense tracker app called
  Spendly, built step-by-step as a tutorial/assignment.

  What it's building:
  - A web app to track personal expenses
  - User authentication (register, login, logout)
  - Expense management (add, edit, delete expenses)
  - User profiles
  - SQLite database backend

  Current state (early stage):
  - Landing, login, and register pages are rendered
  - Most routes are placeholders ("coming in Step X")
  - The database layer (db.py) is empty — students are supposed to implement
  get_db(), init_db(), and seed_db() in Step 1

  The comments reference steps (1–9), so it's structured as a guided tutorial
  where features get implemented incrementally.

❯ what tech stack does this project use                                         

⏺ Backend
  - Python + Flask (web framework)
  - SQLite (database)

  Frontend
  - HTML/Jinja2 templates
  - CSS (style.css)
  - JavaScript (main.js)

  Testing
  - pytest + pytest-flask

  No ORM — raw SQLite connections via a custom db.py helper.
